## 개선 방안 (확정안)

### 1️⃣ 게시판 답변 검수 자동화 (우선순위: 높음 / 현재 보류)

**목표**: NotebookLM 수동 작업을 완전 자동화하여 최종 검증만 남기기

**현재 프로세스**:
- 어드민 페이지에서 Excel 다운로드 (번호, 날짜, 인입경로, 문의유형, 문의내용, 답변내용)
- NotebookLM에 수동 입력 → 검수 → 결과 복붙 → 수동 확인 (총 6시간)
- 수정된 답변을 어드민 페이지에 1건씩 직접 등록

**개선 방식**: 룰 기반 자동화 스크립트로 Excel 직접 처리

**세부 계획**:
```
1. Ver1.1.md 규칙 파싱 → 검수 엔진 구축
   - 오탈자 가능성 (한글 맞춤법 검사: kiwi)
   - 용어 표기 통일 ("잇올앱", "등원 인정" 등)
   - 표현력 개선 (자연스러운 문장 구조)
   - 센터명 통일 (Ver1.1의 100+ 목록 활용)
   - 형식 규칙 (센터 첫 문장, 명칭 통일)

2. 입력 처리 흐름
   - 입력: 어드민 페이지 Excel 파일 (500건)
   - 처리:
     ✓ 각 행의 답변내용 검수
     ✓ Ver1.1 규칙 자동 적용
     ✓ 수정본 생성
     ✓ 변경점 요약 (1줄 1변경, 요약 규칙 준수)

3. 출력 형식 (Ver1.1 준수)
   - 출력 Excel 컬럼: [기존] + [수정답변] + [변경점요약]
   - 수정본: 코드블록 형식
   - 변경점: 띄어쓰기/문장부호 제외, 핵심 변경만

4. 검증 단계
   - 사용자가 수정본 Excel 검토 (최종 확인)
   - 어드민 페이지에 수정답변 복사-등록 (Playwright 자동화 추후 검토)
```

**기술 스택**: Python (kiwi 맞춤법 검사, pandas, openpyxl)
**입출력**: 어드민 페이지 Excel ↔ 검수 결과 Excel
**저장소**: `~/workspace/my-first-ai/` 또는 별도 위치 결정
**예상 효과**: 6시간 → 1시간 (자동 검수 + 사용자 최종확인)

**보류 사유**: 어드민 페이지 샘플 Excel 및 URL 정보 미확보. 재개 조건:
- [ ] 어드민 페이지 Excel 샘플 파일 (5~10건) 제공
- [ ] (Playwright 자동 등록 원할 시) 어드민 페이지 URL + 로그인 방식 공유

### 2️⃣ 크롤링 데이터 필터링 강화 (우선순위: 중 / Phase B)

**목표**: 2차 수동 검증 시간 단축 (현재 1시간 → 30분 이하)

**구현 방식**: 기존 Apps Script 보완 + Python 사전 필터링

**세부 계획**:
```
1. 기존 Apps Script 개선 (간단)
   - NEGATIVE_KEYWORDS 확대 (데이터 패턴 분석 후)
   - POSITIVE_KEYWORDS 우선순위 반영
   - 문장 길이 기준 조정 (현재 10자 → ?)
   - 중복/스팸 필터 추가

2. Python 크롤러 단계에서 사전 필터링 (옵션)
   - 크롤링 후 CSV 생성 전 필터링
   - Apps Script로 보내기 전에 "확실한 쓰레기" 제거
   - 회색 지대만 2차 검증으로

3. 2차 검증 프로세스 최소화
   - 현재: 휴지통의 모든 항목 일일이 확인
   - 개선: 상위 N개만 스팟 체크 (샘플 검증)
   - 또는: 자동 분류 신뢰도 점수 제시 (역순 정렬)
```

**기술 스택**: 기존 dcinside_crawler.py, orbi_crawler.py 활용
