> 작성일: 2026년 3월 14일
> 버전: v0.1 (초안)

---

## 1. 사업 개요

### 서비스 한 줄 소개

**"지금 내 근처에서 영업 중인 포장마차를 바로 찾아주는 앱"**

### 배경 및 문제 정의

겨울에 붕어빵, 어묵, 떡볶이가 먹고 싶을 때 — 혹은 퇴근길에 문득 길거리 음식이 당길 때, 우리는 근처 포장마차가 어디 있는지 찾을 방법이 없다.

- 네이버지도, 카카오맵 등 기존 지도 앱에는 포장마차 정보가 거의 없음
- 배달의민족, 요기요 등 배달 앱은 정식 음식점 위주
- 포장마차는 위치가 유동적이고, 영업 시간이 불규칙하여 기존 플랫폼이 다루기 어려운 영역
- 운영자 입장에서도 본인의 가게를 알릴 수 있는 전용 채널이 없음

### 해결책

포장마차·길거리 음식에 특화된 **위치 기반 탐색 + 리뷰 플랫폼**을 제공한다.

- **소비자**: 앱을 열면 지도에서 지금 영업 중인 포장마차를 바로 확인
- **운영자**: 앱을 켜두면 GPS로 위치가 자동 노출되어 별도 조작 없이 홍보

## 2. 시장 분석

### 시장 규모

| 구분 | 수치 |
|------|------|
| 국내 길거리 음식 시장 규모 (추정) | 약 3~5조 원 |
| 전국 포장마차·노점상 수 (추정) | 약 20~30만 개소 |
| 국내 음식점 리뷰 앱 이용자 수 | 약 2,000만 명+ (배민, 캐치테이블, 망고플레이트 등) |

### 경쟁 환경

| 기존 서비스 | 한계점 |
|------------|--------|
| 네이버지도 / 카카오맵 | 포장마차 정보 미등록, 유동 위치 반영 불가 |
| 배달의민족 / 요기요 | 정식 사업자 위주, 포장마차 진입 장벽 높음 |
| 인스타그램 / 블로그 | 실시간 영업 여부 확인 불가, 검색 비효율 |
| 붕어빵 지도 (시즌 앱) | 붕어빵 한정, 리뷰 기능 부재, 시즌 한정 |

### 차별화 포인트

1. **포장마차 전용** — 기존 앱이 다루지 않는 사각지대에 집중
2. **GPS 실시간 위치** — 운영자가 앱만 켜면 자동으로 위치 노출, 수동 조작 불필요
3. **지금 영업 중 필터** — 검색하는 순간 영업 중인 곳만 표시
4. **리뷰 및 메뉴 정보** — 포장마차에 대한 신뢰도 확보

## 3. 서비스 구성

### 3-1. 소비자 기능

| 기능 | 설명 | MVP 포함 |
|------|------|----------|
| 지도 탐색 | 현재 위치 기반으로 주변 포장마차 핀 표시 | ✅ |
| 영업 중 필터 | GPS 활성화된 (지금 영업 중인) 포장마차만 표시 | ✅ |
| 상세 정보 | 메뉴, 사진, 평점, 영업 패턴 확인 | ✅ |
| 리뷰 작성 | 방문 후 메뉴별 평가 및 한줄평 | ✅ |
| 즐겨찾기 | 자주 가는 포장마차 저장 | ⬜ (이후) |
| 오픈 알림 | 즐겨찾기 포장마차가 영업 시작하면 알림 | ⬜ (이후) |

### 3-2. 운영자 기능

| 기능 | 설명 | MVP 포함 |
|------|------|----------|
| 가게 등록 | 상호명, 메뉴, 사진, 가격 입력 | ✅ |
| GPS 위치 공유 | 앱 켜면 자동 위치 노출, 끄면 비노출 | ✅ |
| 리뷰 확인 | 내 가게에 달린 리뷰 조회 | ✅ |
| 영업 통계 | 방문자 수, 조회 수 등 간단한 대시보드 | ⬜ (이후) |

### 3-3. 플랫폼 유형

- **고정형 포장마차**: 등록된 주소 기반, GPS 공유 선택 사항
- **이동형 포장마차 / 푸드트럭**: GPS 실시간 공유 필수

## 4. 기술 계획

### 기술 스택 (예정)

| 구분 | 기술 | 선정 이유 |
|------|------|-----------|
| 프론트엔드 (앱) | Flutter | Android 우선, 향후 iOS 확장 용이 |
| 백엔드 | Firebase (Firestore, Auth, Cloud Functions) | 서버리스, 무료 티어 충분, 빠른 개발 |
| 지도 | Google Maps API | Flutter와 통합 우수, 한국 지도 데이터 정확 |
| 위치 추적 | GPS + Geolocation API | 운영자 실시간 위치 공유 |
| 이미지 저장 | Firebase Storage | 메뉴/리뷰 사진 업로드 |

### 개발 방식

- LLM(Claude, ChatGPT 등)을 적극 활용한 코드 생성 및 학습 병행
- 개발 경험 없지만 간단한 코드(Apps Script, 크롤링)를 LLM으로 작성해본 경험 보유

### 개발 일정 (목표: 2~3개월)

| 주차 | 마일스톤 |
|------|----------|
| 1~2주 | 기획 완료: 앱 이름, 브랜딩, 화면 설계 (와이어프레임) |
| 3주 | 개발 환경 세팅: Flutter, Firebase, Google Play 계정 |
| 4~6주 | 핵심 기능 개발: 지도 화면, 운영자 GPS, 가게 등록 |
| 7~9주 | 리뷰 기능, 상세 화면, 회원가입/로그인 |
| 10~11주 | 테스트, 버그 수정, 디자인 다듬기 |
| 12주 | Google Play 출시 |

## 6. 성장 전략

### 6-1. 초기 데이터 확보 (콜드 스타트 해결)

앱이 비어있으면 아무도 쓰지 않는다. 출시 전후 초기 데이터 확보가 핵심.

1. **직접 등록**: 본인 동네 포장마차를 직접 찾아가 사진 촬영 + 정보 등록
2. **운영자 직접 모집**: 포장마차 사장님에게 직접 설명하고 앱 설치 유도
3. **시즌 공략**: 겨울 시즌(붕어빵, 호떡, 어묵) 직전 출시 → 수요가 가장 높은 시기에 런칭

### 6-2. 마케팅 채널

| 채널 | 방법 |
|------|------|
| SNS (인스타그램, 틱톡) | 포장마차 탐방 콘텐츠, "오늘의 포장마차" 시리즈 |
| 커뮤니티 (에브리타임, 블라인드) | 겨울 시즌 "근처 붕어빵 어디있어요" 게시글에 앱 소개 |
| 포장마차 현장 | 참여 운영자에게 스티커/배너 제공 → 오프라인 바이럴 |
| 지역 언론 / 블로거 | 앱 소개 기사 및 체험 리뷰 |

### 6-3. 확장 로드맵

| 단계 | 목표 | 기간 |
|------|------|------|
| Phase 1 | 특정 지역(서울 1~2개 구) 집중 | 출시 후 1~3개월 |
| Phase 2 | 수도권 확대 + iOS 출시 | 3~6개월 |
| Phase 3 | 전국 확대 + 프리미엄 기능 | 6~12개월 |
