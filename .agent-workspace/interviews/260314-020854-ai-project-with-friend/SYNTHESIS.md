# 포장마차 앱 - Synthesis / 종합

## What This Is / 개요

포장마차·길거리 음식을 먹고 싶을 때 주변에서 찾기 어렵다는 직접적인 불편함에서 출발한 앱 프로젝트. 친구와 둘이서 LLM을 활용해 Android 앱을 제작하며, AI 기술 학습과 실제 서비스 출시라는 두 가지 목표를 동시에 추구한다.

## Key Insights / 핵심 인사이트

1. **명확한 문제 정의**: "겨울에 붕어빵·어묵·떡볶이가 먹고 싶은데 어디 있는지 찾을 수가 없다" — 직접 겪은 불편함에서 나온 아이디어라 동기가 강하다.
2. **양면 플랫폼 구조**: 소비자(근처 포장마차 찾기+리뷰)와 운영자(GPS로 자동 홍보) 양쪽 모두에게 가치가 있다.
3. **GPS 실시간 공유가 핵심**: 운영자가 앱을 켜두면 위치가 자동 노출되는 방식. "오늘 영업" 버튼도 결국 수동이라 GPS 실시간이 더 자연스럽다.
4. **기존 앱의 공백**: 배달앱·네이버지도 등에서 포장마차는 제대로 다뤄지지 않는 영역 — 틈새시장이 존재한다.
5. **LLM 활용 개발**: 개발 경험 없이도 Apps Script·크롤링 등을 LLM으로 만들어본 경험 있음. 같은 방식으로 앱 개발도 도전 가능.

## Constraints & Boundaries / 제약사항 및 경계

- 정식 개발 경험 없음 (LLM 보조 개발로 진행)
- 2~3개월 내 제작 완료 목표 (2026년 5~6월)
- Android 먼저, iOS는 추후
- AI 기능 탑재 없이 순수 앱 서비스로 출발
- 사이드 프로젝트 — 둘 다 현직 재직 중

## Decisions / 결정사항

- **앱 형태**: Android 앱 (웹 아님)
- **위치 방식**: 운영자 GPS 실시간 공유
- **역할 분담**: 기획·개발·디자인 둘이 함께
- **AI 활용 여부**: 서비스 내 AI 기능 없음, 개발 도구로만 LLM 활용
- **타겟 사용자**: 20~60대, 성별 무관 / 핵심 사용 순간은 "지금 당장 근처 포장마차를 찾고 싶을 때"

## Approach / Direction / 접근방식 및 방향

**핵심 경험**: 앱을 열면 지금 내 주변에서 영업 중인 포장마차가 지도에 바로 보인다.

- 소비자 앱: 지도 기반 주변 포장마차 탐색 → 메뉴·리뷰 확인
- 운영자 앱(또는 별도 화면): GPS 켜면 자동 위치 공유 → 마감 시 종료
- 리뷰 기능: 방문 후 메뉴·맛 평가 남기기

MVP는 이 세 가지에 집중하고, 부가 기능은 출시 후 추가.

## What's Deferred / 보류된 항목

- 수익화 모델 (광고, 운영자 유료 플랜 등)
- 웹 버전
