📖 지난 편 복습: 세 AI의 핵심만 기억하세요

1편에서 AI의 기본 개념을, 2편에서 ChatGPT·Gemini·Claude의 차이를 살펴봤습니다. 오늘은 그 지식을 바탕으로 실제 업무에서 어떻게 쓰는지 집중적으로 다뤄보겠습니다.

ChatGPT  → 📣 팔방미인, 이미지 생성, 다양한 도구 연동
Gemini    → 🔍 실시간 정보, 구글 생태계와 딱
Claude    → 📖 글쓰기 최강, 긴 문서 분석, 신뢰도 높음

🤔 왜 같은 AI인데 누구는 "대박이다!", 누구는 "별로다"를 외칠까요?

AI를 처음 쓰는 분들이 가장 많이 하는 실수는 바로 너무 짧게 묻는 것입니다. AI는 당신의 상황, 목적, 요구사항을 알 수가 없습니다. 내가 많이 알려줄수록, AI는 그만큼 더 정확하고 유용한 답을 줍니다.

❌ 이렇게 물으면

"보고서 써줘"

→ 어떤 주제인지, 누가 읽는지, 분량은 얼마인지 모르니 뻔한 결과만 나옵니다.

✅ 이렇게 물으면

"나는 IT 스타트업 마케팅 팀장이야. CFO께 Q1 마케팅 실적을 보고하는데, 핵심 수치 3가지와 다음 분기 계획을 담아서 2페이지로 써줘."

→ 상황, 독자, 형식까지 알려주면 실무에서 바로 쓸 수 있는 결과가 나옵니다.

🎯 프롬프트 잘 쓰는 4가지 공식

복잡하게 생각할 필요 없습니다. 아래 4가지 요소만 챙겨도 결과물이 확 달라집니다.

요소 설명 예시
① 역할 부여 AI에게 어떤 전문가 역할을 맡길지 알려주기 "너는 10년 경력의 마케터야"
② 배경 설명 현재 상황, 맥락, 독자 정보 제공하기 "우리 회사는 B2B SaaS이고, 보고 대상은 임원이야"
③ 구체적 요청 무엇을, 어떻게, 얼마만큼 원하는지 명확히 "핵심 이점 3가지를 bullet point로, 300자 이내로"
④ 조건 추가 피해야 할 것, 반드시 포함할 것 명시 "전문 용어는 피하고, 구체적 수치는 꼭 넣어줘"

💡 꿀팁: 4가지를 한 번에 다 담으려 하지 않아도 됩니다. 일단 보내고, 결과가 마음에 안 들면 "이 부분을 이렇게 바꿔줘"라고 이어서 요청하면 됩니다. AI와의 대화는 한 번의 질문으로 끝내는 것이 아니라 핑퐁처럼 주고받는 과정입니다.

💼 직장 상황별 실전 프롬프트 예시

아래 예시들은 실제 직장인의 업무 상황에서 바로 복사해서 쓸 수 있도록 작성했습니다. 각 상황에 어떤 AI가 더 적합한지도 함께 안내합니다.

✉️ 상황 1: 비즈니스 이메일 작성

추천 AI: Claude (자연스러운 문체 최강) / ChatGPT (다양한 톤 변주 가능)

이메일은 직장인이 AI에게 가장 많이 요청하는 작업 중 하나입니다. 하지만 "이메일 써줘"라고만 하면 누구에게 보내는지, 어떤 목적인지를 모르니 뻔한 결과가 나옵니다.

📝 신규 고객사 제안 이메일 예시

너는 IT 스타트업의 영업 담당자야. 친근하면서도 전문적인 어투를 써. 상황: 500~1000명 규모의 금융회사 HR 담당자에게 우리 팀 협업 플랫폼(Slack의 상위 버전)을 처음 소개하는 이메일이야. 요청: 2주 무료 트라이얼을 제안하고, 미팅 일정을 잡자는 내용으로 마무리해줘. 조건: - 200자 이내로 짧고 간결하게 - 존댓말로, 친근하지만 신뢰감 있게 - 과도한 세일즈 표현(예: "놀라운", "혁신적인")은 피해줘

📝 까다로운 고객의 우려 사항 회신 예시

고객사 담당자가 이런 메일을 보냈어: "좋은 제안인데, 저희는 금융회사라 보안이 최우선이에요. ISO 27001 인증이 있나요?" 이 메일에 대한 회신을 작성해줘. - 우리 회사는 ISO 27001 인증 보유 (2024년 8월 취득) - SOC 2 Type II 감사 진행 중 (2025년 완료 예정) - 우려를 충분히 공감하는 어조로 시작 - 자랑스럽게 늘어놓지 말고, 신뢰를 쌓는 방향으로 - 다음 단계로 보안 담당자 1:1 미팅 제안으로 마무리

📊 상황 2: 보고서 · 기획서 작성

추천 AI: Claude (긴 문서 맥락 유지 탁월, 논리 구조 잡기 최강)

보고서나 기획서는 분량이 길고 논리 흐름이 중요합니다. Claude는 이 작업에서 특히 빛을 발합니다. 긴 초안도 일관된 톤을 유지하며 끝까지 잘 써줍니다.

📝 분기 실적 보고서 초안 요청 예시

너는 5년 차 마케팅 팀장이야. Q1 2026년 마케팅 실적 보고서를 작성해줘. 보고 대상은 우리 회사 CFO야. 회사 소개: B2B HR SaaS 스타트업, 팀원 50명 보고서 구성: 1. Executive Summary (핵심 수치 3개 + 한 줄 인사이트, 1페이지) 2. 채널별 성과 분석 (이메일 / SNS / 유료 광고 각각) 3. CAC(고객 획득 비용) 분기 추이와 해석 4. 다음 분기 우선 액션 3가지 (이유 포함) 조건: - 숫자는 현실적인 예시 값으로 채워줘 - CFO가 보는 거니까 핵심만, 군더더기 없이 - 각 섹션은 소제목으로 구분해줘

📝 보고서 피드백 받기 예시

아래는 임원 보고용 기획서 초안이야. 다음 4가지 기준으로 검토해줘. 1. 논리 흐름: 핵심 메시지가 처음부터 끝까지 일관된가? 2. 데이터 설득력: 주장을 뒷받침하는 수치가 충분한가? 3. 실행 가능성: 제안한 액션이 현실적인가? 4. 독자 친화성: 임원이 빠르게 읽기 편한 구성인가? 각 항목마다 점수(5점 만점)와 구체적인 개선 방향을 알려줘. [여기에 기획서 내용 붙여넣기]

🔍 상황 3: 데이터 정리 & 분석

추천 AI: Claude (대용량 데이터 분석) / Gemini (최신 경쟁사 동향 파악)

대량의 고객 피드백, 설문 결과, 시장 데이터를 정리하는 일은 원래 몇 시간이 걸립니다. AI를 쓰면 5분 만에 핵심 인사이트를 뽑아낼 수 있습니다.

📝 고객 피드백 분석 예시

아래는 고객 만족도 설문 응답 100건이야. 이 데이터를 분석해서: 1. 긍정 피드백 vs 부정 피드백 비율 2. 가장 많이 언급된 주제 상위 5개 (빈도 포함) 3. 만족도 5점 고객 vs 1~2점 고객의 차이점 4. 즉시 개선이 필요한 영역 3가지 (이유 포함) 5. 팀에 공유할 한 줄 요약 을 정리해줘. 가능하면 표 형식으로. [여기에 설문 응답 데이터 붙여넣기]

📝 경쟁사 동향 분석 예시 (Gemini 추천)

최근 3개월간 [경쟁사 A], [경쟁사 B], [경쟁사 C]의 움직임을 조사해서 아래 항목으로 정리해줘. - 새롭게 출시한 기능이나 서비스 - 마케팅 캠페인이나 이벤트 - 가격 정책 변화 - 채용 공고 방향 (조직 확장 힌트) - 언론 보도나 펀딩 소식 표로 정리하고, 마지막에 우리 회사가 주목해야 할 점을 2~3개 짚어줘.

💡 상황 4: 회의 준비 & 아이디어 발굴

추천 AI: ChatGPT (다양한 아이디어 발산) / Claude (깊이 있는 논리 구조화)

막힌 상황에서 혼자 끙끙 앓기보다, AI를 브레인스토밍 파트너로 활용해보세요. 빠르게 아이디어를 쏟아내고, 그 중에서 좋은 것을 골라 발전시키는 방식이 효율적입니다.

📝 문제 해결 브레인스토밍 예시

우리 팀이 지금 이런 문제를 겪고 있어: "고객 이탈률이 지난 분기 대비 15% 증가했고, 이탈 고객의 50%가 온보딩 30일 이내에 발생한다." 이 문제를 해결할 수 있는 아이디어를 10가지 제안해줘. - 비용이 거의 들지 않는 방법도 포함해줘 - 각 아이디어마다 예상 효과와 실행 난이도(상/중/하)도 알려줘 - 창의적인 접근법과 이미 검증된 방법을 섞어서 제안해줘

📝 의사결정 비교 분석 예시

우리가 두 가지 마케팅 채널 중 하나를 선택해야 해. 옵션 A: LinkedIn 유료 광고 옵션 B: 콘텐츠 마케팅 (블로그 + 뉴스레터) 우리 상황: - B2B SaaS, 타깃은 기업 HR 담당자 - 월 예산 300만 원 - 팀 인원 2명, 빠른 성과보다 장기 자산 구축 선호 두 옵션을 이 기준으로 비교해줘: 초기 비용, 결과까지 걸리는 시간, 장기적 자산 가치, 팀 리소스 부담. 그리고 우리 상황에 어떤 걸 추천하는지, 이유와 함께 명확하게 말해줘.

💻 상황 5: 코딩 & 업무 자동화

추천 AI: Claude (복잡한 로직·설명) / ChatGPT Codex (코드 생성·디버깅)

개발자가 아니어도 괜찮습니다. 반복적인 Excel 작업, 데이터 변환, 간단한 자동화 스크립트 등은 AI가 충분히 도와줄 수 있습니다.

📝 코드 버그 수정 요청 예시

아래 파이썬 코드가 있는데, 실행하면 예상과 다른 결과가 나와. 어떤 문제가 있는지 찾아주고: 1. 문제가 되는 부분과 이유를 한국말로 설명해줘 2. 수정된 코드를 보여줘 3. 앞으로 비슷한 실수를 피하려면 어떻게 해야 하는지도 알려줘 [여기에 코드 붙여넣기]

📝 비개발자용 자동화 요청 예시

나는 개발자가 아니야. 매주 이런 작업을 수동으로 하고 있어: "엑셀 파일에서 '완료' 상태인 행만 골라 새 파일로 저장하고, 날짜별로 정렬하기" 이 작업을 자동화하는 파이썬 스크립트를 만들어줘. - 코드를 전혀 모르는 사람도 따라할 수 있도록 실행 방법을 단계별로 설명해줘 - 필요한 프로그램 설치 방법도 포함해줘

🎛️ 상황별 AI 선택 가이드 (한눈에 보기)

이런 상황이라면 추천 AI 이유
자연스러운 글쓰기, 이메일, 보고서 Claude 문맥 이해 + 자연스러운 한국어 표현
최신 뉴스, 시장 동향, 실시간 정보 Gemini 구글 검색 연동으로 실시간 데이터 반영
이미지 생성, 시각 자료 제작 ChatGPT DALL-E 통합으로 텍스트→이미지 바로 생성
긴 PDF, 계약서, 보고서 통째 분석 Claude 100만 토큰 컨텍스트로 책 한 권도 소화
다양한 아이디어 브레인스토밍 ChatGPT 창의적 발산 능력 + 다양한 관점 제시
복잡한 코드 분석, 버그 찾기 Claude 논리 추론 능력으로 오류 원인 정확히 짚음
Gmail 초안, 구글 드라이브 파일 분석 Gemini 구글 서비스와 직접 연동
신뢰성 중요한 정보, 솔직한 피드백 Claude 모르면 모른다고 말하는 솔직한 성향

📋 즉시 활용 가능한 프롬프트 템플릿 5가지

아래 템플릿에서 [ ] 안의 내용만 바꿔서 사용하면 됩니다. 붙여넣기 후 바로 시작할 수 있어요.

📌 템플릿 1: 문서 초안 작성

너는 [역할/직책]이야. [독자/상황]을 위한 [문서 종류]를 작성해줘. 포함할 내용: - [핵심 내용 1] - [핵심 내용 2] - [핵심 내용 3] 조건: - 분량: [분량] - 톤: [정중한/친근한/전문적인] - 특별히 강조할 점: [강조 사항] - 피해야 할 표현: [금지 표현]

📌 템플릿 2: 데이터 분석 및 인사이트 도출

아래 [데이터 종류]를 분석해줘. 분석 항목: 1. [분석 항목 1] 2. [분석 항목 2] 3. 패턴이나 특이점이 있다면 짚어줘 4. 이 데이터를 바탕으로 추천 액션 3가지 형식: 표로 정리하고, 마지막에 한 줄 요약 추가 [데이터 붙여넣기]

📌 템플릿 3: 문제 해결 아이디어 발굴

우리 팀이 [문제]를 겪고 있어. 배경: - [상황 1] - [상황 2] - 주요 제약: [예산/시간/인력 제약] 이 문제를 해결할 아이디어 10가지를 제안해줘. 각각 예상 효과(높음/중간/낮음)와 실행 난이도(쉬움/보통/어려움)도 함께 알려줘. 비용이 거의 없는 방법도 꼭 포함해줘.

📌 템플릿 4: 복잡한 개념 쉽게 배우기

[개념]을 [나의 직업/업무]를 하는 사람에게 설명하듯 알려줘. 구성: 1. 한 문장 정의 (전문 용어 없이) 2. 내 업무와 연결되는 실제 예시 3. 왜 알아야 하는가? 4. 흔히 오해하는 부분 5. 더 깊이 공부하고 싶다면 어디서 시작하면 좋을까?

📌 템플릿 5: 피드백 루프 (초안 개선)

# 1단계: 초안 요청 "[문서 종류]의 초안을 만들어줘. [간단한 요구사항]" # 2단계: 특정 부분 개선 요청 "초안은 좋은데, [개선하고 싶은 부분]이 좀 아쉬워. [이렇게] 바꿔줄 수 있어?" # 3단계: 최종 마무리 "이제 거의 완성됐어. [체크할 항목]만 한 번 더 확인해주고, [특정 기준]에 맞게 마지막으로 다듬어줘."

⚠️ 반드시 알아야 할 주의사항 3가지

① 민감한 정보는 절대 넣지 마세요

거래처 정보, 고객 개인정보, 미공개 프로젝트 내용, 내부 재무 수치 등은 AI에 입력하지 마세요. 입력한 내용이 AI 학습에 활용될 수 있습니다. 민감한 정보는 가명 처리 또는 일반화해서 사용하세요. (예: "삼성전자" → "대기업 전자 회사")

② AI가 자신 있게 틀릴 수 있습니다 (할루시네이션)

AI는 없는 사실을 있는 것처럼 매우 자신 있게 말하는 경우가 있습니다. 특히 구체적인 숫자, 날짜, 고유명사, 법률/규정 정보는 반드시 원문 출처에서 검증하세요. AI 결과물은 항상 초안으로만 사용하고, 최종 판단은 반드시 사람이 해야 합니다.

③ AI가 "어떻게"를 잘한다면, "무엇을"은 사람이

AI는 주어진 작업을 빠르게 처리하는 데 능숙합니다. 하지만 어떤 문제를 풀어야 하는지는 여전히 사람의 몫입니다. AI에게 방향을 맡기지 말고, 방향은 내가 잡고 실행을 AI에게 맡기는 구조로 활용하세요.
